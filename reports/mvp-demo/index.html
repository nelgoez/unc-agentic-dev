Apéndice Técnico
Cómo funciona el JSON de disponibilidad

Moodle almacena las restricciones de acceso en un campo JSON dentro de la tabla course_modules. Cada restricción sigue este formato:

{
  "type": "completion",
  "cm": 6628,
  "e": 1
}

Donde:

  • type: "completion" (requiere completar una actividad) o "grade" (requiere nota mínima)
  • cm: ID del course module que debe estar completado
  • e: 1 = marcado como completado, 2 = cualquier estado de completado
  • id: (para tipo grade) ID del grade item
  • min: (para tipo grade) nota mínima requerida
Cómo persisten los phantoms

Cuando una actividad es eliminada en Moodle, el sistema no limpia automáticamente las restricciones que la referencian. El JSON de disponibilidad persiste en la base de datos, resultando en una actividad fantasma: una restricción que referencia un cmid que ya no existe. Esto puede causar bloqueos permanentes para estudiantes, ya que la condición de completitud nunca podrá satisfacerse.

Metodología de detección
  1. Obtener estructura del curso vía core_course_get_contents
  2. Extraer JSON de disponibilidad de cada módulo
  3. Detectar cmids referenciados que no existen en la estructura actual (findOrphanedCmIds())
  4. Crear usuario estudiante nuevo vía API REST
  5. Matricular el usuario en el curso
  6. Obtener datos de completación como estudiante real
  7. Comparar con datos de nelthor (admin) para identificar discrepancias
  8. Eliminar el usuario de prueba
